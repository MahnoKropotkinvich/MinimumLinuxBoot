# Generic MinimumLinuxBoot for RTL Simulations
GSoC 2026 Project [Generic MinimumLinuxBoot for RTL Simulations](https://summerofcode.withgoogle.com/programs/2026/projects/TBGVWAn6). This project aims to drastically reduce total time spent on waiting for booting when running Linux on [OpenPiton](https://github.com/PrincetonUniversity/OpenPiton) on [Verilator RTL emulator](https://github.com/verilator/verilator).
## Prerequisites
- **build dependencies** for Verilator, qemu and Linux kernel, it is usually accessible from your package manager (`dnf builddep qemu kernel verilator` in Fedora/RHEL, for example) or you can manually install deps according to upstream
- **OpenPiton Verilator C++ emulator**, which can be generated by its `sims` utility (you shouldn't change its memory image schema, since this tool only generates the original one)
- **native or cross RISC-V build toolchain**, cross toolchain is accessible from https://github.com/riscv-collab/riscv-gnu-toolchain (**both newlib and Linux**)
- **curl**, to download alpine rootfs

## Components
- a forked version of **QEMU**, providing `OpenPiton` RISC-V machine type to match exactly real hardware's peripheral addresses
- **Linux kernel**, no changes to source code but configurations, it currently **only** supports initramfs and uart interaction.
- **OpenSBI**, submodule from upstream, no changes
- **Python scripts**, for state capture & image schema conversion 
- **M-mode stub**, to restore all saved states in memory image
- **ebreak trigger**, serving as the checkpoint for gdb to capture memory state 
- **Makefile**, for toolchain building and automated pipeline
## Usage
1. Ensure all prerequisites are met
1. Init all submodules
2. Automatically build toolchain and capture designated memory image, may need to specify where to find cross toolchain(`riscv64-unknown-linux-gnu-gcc`, you can its name)
```bash
make PREFIX=/path/to/cross capture -j$(nproc)
```
or use your custom initramfs (QEMU recognizable format)
```bash
make PREFIX=/path/to/cross INITRD=/path/to/initramfs.cpio capture -j$(nproc)
```
The provided initramfs **MUST** run ebreak trigger at checkpoint state. For example, in Alpine inittab
```config
/etc/inittab:
::once:/ebreak_trigger
```
3. The output dir is under `build/`, which includes both toolchain binaries and memory image. The converted memory image is `build/mem.image`
## Notes
- OpenPiton itself is not shipped with this repo, you can build it on your own. 
- OpenPiton does not provide an interactive fake UART device either but a read-only one. You can manually modify it into a NS16550-compatible read-write device.
